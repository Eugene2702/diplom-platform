# DiplomaHub — нагрузочное тестирование

Соответствует **таблице 4.2** главы 4 ВКР: измеряет среднее время ответа,
P95 и P99 для 5 наиболее часто вызываемых эндпоинтов под нагрузкой
50 → 100 виртуальных пользователей.

## Инструмент

[Artillery 2.x](https://www.artillery.io/) — заявлен в ВКР, установлен как
`devDependency` бэкенда (`npm install` подтянет автоматически).

## Запуск

```powershell
# 1. Поднять весь стек
docker compose up -d

# 2. Подготовить тестовые данные (создаёт юзера, проект, 200 коммитов и .env)
docker compose exec backend npm run load:seed

# 3. Прогон под 50 одновременных пользователей (целевой по ВКР)
docker compose exec backend npm run load:test

# 4. (опционально) пиковая фаза — 100 пользователей
docker compose exec backend npm run load:test:peak
```

Artillery печатает финальный отчёт в стандартный вывод. Сводка содержит
`http.response_time` со средним, медианой, p95, p99, max — это и есть
исходники для таблицы 4.2.

## Фазы нагрузки

| Фаза   | Длительность | Скорость прибытия       |
|--------|--------------|--------------------------|
| warm-up | 30 с        | 10 RPS (прогрев пулов)   |
| ramp    | 30 с        | 10 → 50 RPS              |
| steady  | 60 с        | 50 RPS (замерочный отр.) |

## Эндпоинты и веса

| Сценарий                                | Weight | Комментарий                              |
|------------------------------------------|--------|------------------------------------------|
| `GET /api/projects`                      | 35     | список (основной read-path)              |
| `GET /api/projects/:id`                  | 25     | детальная страница                       |
| `POST /api/auth/login`                   | 15     | bcrypt 12 раундов — самый тяжёлый read   |
| `GET /api/projects/:id/git/stats`        | 20     | агрегация по таблице commits             |
| `POST /api/projects/:id/git/sync`        | 5      | внешний HTTP к GitHub                    |

> Низкий вес `git/sync` — намеренно. GitHub API ограничивает 5000 запросов/час
> с токеном и 60/час без него; при weight≥20 тест упёрся бы в rate-limit и
> исказил бы метрики остальных эндпоинтов.

## Авторизация

`load:seed` генерирует JWT и пишет его в `backend/load-tests/.env`.
`processor.js` подгружает этот файл и инжектит переменные в каждого
виртуального пользователя.

## Пороги (config.ensure.thresholds)

Если p95 > 500 мс или p99 > 1000 мс — Artillery вернёт ненулевой exit code.
Эти пороги совпадают с заявленным в ВКР SLA «менее 200 мс среднее, до
200–480 мс p95 при 100 RPS».
